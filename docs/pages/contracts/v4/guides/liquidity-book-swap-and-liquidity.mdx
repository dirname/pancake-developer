# 流动性和流动性书池的交换

在本指南中，我们将使用 [v4-periphery](https://github.com/pancakeswap/pancake-v4-periphery) 的外围合约

1. 通过 `BinFungiblePositionManager` 进行流动性操作
2. 通过 `BinSwapRouter` 进行交换

<Callout type="info">
  在调用下面的任何函数之前，请确保已批准 BinFungiblePositionManager 或 BinSwapRouter。
</Callout>

## 流动性操作

[BinFungiblePositionManager](https://github.com/pancakeswap/pancake-v4-periphery/blob/main/src/pool-bin/BinFungiblePositionManager.sol)

### 增加流动性

```solidity
IBinFungiblePositionManager.AddLiquidityParams memory params = IBinFungiblePositionManager
  .AddLiquidityParams({
    poolKey: key,
    amount0: amountX,
    amount1: amountY,
    amount0Min: 0, // 注意，这个值在实际应用中不应为0
    amount1Min: 0, // 注意，这个值在实际应用中不应为0
    activeIdDesired: uint256(currentActiveId),
    idSlippage: 0,
    deltaIds: convertToRelative(binIds, currentActiveId),
    distributionX: distribX,
    distributionY: distribY,
    to: address(this),
    deadline: block.timestamp + 600
});

(uint128 amount0, uint128 amount1, uint256[] memory tokenIds, uint256[] memory liquidityMinted)
  = positionManager.addLiquidity(params);

/// @dev 给定 binIds 和 activeIds 列表，返回 delta ids。
//       例如，给定 id: [100, 101, 102] 和 activeId: 101，返回 [-1, 0, 1]
function convertToRelative(uint24[] memory absoluteIds, uint24 activeId)
  internal
  pure
  returns (int256[] memory relativeIds) {
    relativeIds = new int256[](absoluteIds.length);
    for (uint256 i = 0; i < absoluteIds.length; i++) {
      relativeIds[i] = int256(uint256(absoluteIds[i])) - int256(uint256(activeId));
    }
}
```

输入：

- `poolKey`: 用于标识池的结构体
- `amount0 | amount1`: token0 和 token1 的数量
- `amount0Min | amount1Min`: token0 和 token1 的最小数量。如果所需的 token 小于该值，交易将被回滚
- `activeIdDesired`: 增加流动性时所希望的 active id
- `idSlippage`: 最大 active id 滑点。若 activeId 变动超过 idSlippage 值，交易回滚
- `deltaIds`: 用于增加流动性的 delta ids 列表 (`deltaId = activeId - desiredId`)。参见上面的辅助方法 `convertToRelative`
- `distributionX`: token0 的分配，总和（distributionX）为 100e18 (100%)
- `distributionY`: token1 的分配，总和（distributionY）为 100e18 (100%)
- `to`: 接收 1155 token 的收件人的地址，代表流动性的所有权
- `deadline`: 此交易的截止时间。如果 `block.timestamp` 大于 `deadline`，交易将被回滚

输出：
- `amount0`: 用于增加流动性的 token0 数量
- `amount1`: 用于增加流动性的 token1 数量
- `tokenIds`: 为每个 bin 铸造的 tokenId 列表
- `liquidityMinted`: 为每个 bin 铸造的流动性列表

### 移除流动性

```solidity
IBinFungiblePositionManager.RemoveLiquidityParams memory params = IBinFungiblePositionManager
  .RemoveLiquidityParams({
    poolKey: key,
    amount0Min: 0,
    amount1Min: 0,
    ids: ids,
    amounts: amounts,
    from: address(this),
    to: address(this),
    deadline: block.timestamp
});

(uint128 amount0, uint128 amount1, uint256[] memory tokenIds)
  = positionManager.removeLiquidity(params);
```

输入：

- `poolKey`: 用于标识池的结构体
- `amount0Min | amount1Min`: token0 和 token1 的最小数量。如果输出 token 小于此值，交易将被回滚
- `ids`: bin id 列表
- `amounts`: 每个 bin 要移除的流动性数量
- `from`: 持有流动性的持有者地址
- `to`: 接收移除流动性后 token0 和 token1 的用户
- `deadline`: 此交易的截止时间。如果 `block.timestamp` 大于 `deadline`，交易将被回滚

## 交换

[CLSwapRouter](https://github.com/pancakeswap/pancake-v4-periphery/blob/main/src/pool-cl/CLSwapRouter.sol)

### 精确输入单次兑换

```solidity
IBinSwapRouterBase.V4BinExactInputSingleParams memory params = IBinSwapRouterBase
  .V4BinExactInputSingleParams({
    poolKey: key1,
    swapForY: true,
    recipient: address(this),
    amountIn: 1 ether,
    amountOutMinimum: 0,
    hookData: new bytes(0)
});

uint256 amtOut = router.exactInputSingle(params, block.timestamp);
```

输入：

- `poolKey`: 用于标识池的结构体
- `swapForY`: 如果为 true，交换 token0 为 token1，否则交换 token1 为 token0
- `recipient`: 接收输出 token 的地址
- `amountIn`: 所需的输入数量
- `amountOutMinimum`: 最小输出数量。如果输出数量小于该值，交易将回滚
- `hookData`: 如果设置，将在 beforeSwap 和 afterSwap 中传递给 hooks
- `deadline`: 此交易的截止时间。如果 `block.timestamp` 大于 `deadline`，交易将被回滚

输出：

- `amountOut`: 交换得到的输出 token 数量

### 精确输入

```solidity
bytes32 params;
ISwapRouterBase.PathKey[] memory path = new ISwapRouterBase.PathKey[](2);

path[0] = ISwapRouterBase.PathKey({
  intermediateCurrency: currency1,
  fee: uint24(3000),
  hooks: IHooks(address(0)),
  hookData: new bytes(0),
  poolManager: poolManager,
  parameters: params.setBinStep(10)
});

path[1] = ISwapRouterBase.PathKey({
  intermediateCurrency: currency2,
  fee: uint24(3000),
  hooks: IHooks(address(0)),
  hookData: new bytes(0),
  poolManager: poolManager,
  parameters: params.setBinStep(10)
});

IBinSwapRouterBase.V4BinExactInputParams memory input = IBinSwapRouterBase
  .V4BinExactInputParams({
    currencyIn: currency0,
    path: path,
    recipient: address(this),
    amountIn: 1 ether,
    amountOutMinimum: 0
});

uint256 amountOut = router.exactInput(input, block.timestamp);
```

输入：

- `currencyIn`: 输入的 token
- `path`: 描述跳跃路径的 `PathKey[]` 数组 
- `recipient`: 接收输出 token 的地址
- `amountIn`: 所需的输入数量
- `amountOutMinimum`: 最小输出数量。如果输出数量小于该值，交易将回滚
- `deadline`: 此交易的截止时间。如果 `block.timestamp` 大于 `deadline`，交易将被回滚

输出：
- `amountOut`: 交换得到的输出 token 数量
