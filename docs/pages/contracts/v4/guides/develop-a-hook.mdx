# 开发一个 Hook

在本指南中，我们将开发一个用于集中流动性池的 Hook。同样的步骤也适用于流动性书池。我们将从介绍 Hook 模板开始，然后逐步指导。

## Hooks 模板

请访问 https://github.com/pancakeswap/pancake-v4-hooks-template 获取 Hook 模板。点击 `Use this template` 创建一个基于该模板的新仓库。

该模板需要 Foundry。如果你没有安装 Foundry，请按照 [安装指南](https://book.getfoundry.sh/getting-started/installation) 操作。

将新仓库克隆到本地环境后，运行以下命令：
```
> forge install // 安装依赖
> forge test // 运行仓库中的现有测试
```

在 `src` 和 `test` 下各有两个文件夹：`pool-cl` 和 `pool-bin`。如果你在为集中流动性池开发，请专注于 pool-cl 文件夹，否则为流动性书池类型开发请专注于 pool-bin 文件夹。

### BaseHook
BaseHook 是一个基本合约，你可以继承它以创建你的 hooks。它提供

1. 帮助方法： `_hooksRegistrationBitmapFrom` 用于设置所需的回调
2. 回调方法：供你覆盖

```solidity
// 来自 CLCounterHook.sol 的代码片段
import {CLBaseHook} from "./CLBaseHook.sol";

contract CLCounterHook is CLBaseHook {

  constructor(ICLPoolManager _poolManager) CLBaseHook(_poolManager) {}

  // 1. 设置所需的回调。在这种情况下，需要 4 个回调
  function getHooksRegistrationBitmap() external pure override returns (uint16) {
    return _hooksRegistrationBitmapFrom(
        Permissions({
            beforeInitialize: false,
            afterInitialize: false,
            beforeAddLiquidity: true,
            afterAddLiquidity: true,
            beforeRemoveLiquidity: false,
            afterRemoveLiquidity: false,
            beforeSwap: true,
            afterSwap: true,
            beforeDonate: false,
            afterDonate: false,
            noOp: false
        })
    );
  }

  // 2. 对于每个所需回调，覆盖该方法
  function beforeAddLiquidity(address,PoolKey calldata key, ICLPoolManager.ModifyLiquidityParams calldata, bytes calldata)
    external override poolManagerOnly returns (bytes4) {
      // 实现 hook 逻辑后返回选择器
      return this.beforeAddLiquidity.selector;
  }
}

```

## 逐步指南

我们将开发一个 Hook，使 [veCake](https://docs.pancakeswap.finance/products/vecake/what-is-vecake) 持有者在通过带此 Hook 的池进行交换时享受 50% 交换费折扣。

### 步骤 1：下载 Hook 模板

1. 从 pancake-v4-hooks-template 创建一个新仓库：[点击这里](https://github.com/new?template_name=pancake-v4-hooks-template&template_owner=pancakeswap) 
2. 将仓库克隆到本地并运行 `forge install` 和 `forge test` 验证设置。

### 步骤 2：实现思路

流程如下：

1. 在交换之前，CLPoolManager 会调用 `hook.beforeSwap()`。
2. 在 `beforeSwap()` 中，hook 调用 `CLPoolManager.updateDynamicSwapFee()` 来更新交换费。这就是我们的 hook 将交换费更新为 50% 折扣或默认费用的地方。

![高层次](/v4/guides-develop-hook-1.png)

### 步骤 3：实现 Hook

我们将进行以下操作：

1. 添加 `beforeSwap` 权限
2. 在 `beforeSwap()` 中调用 `poolManager.updateDynamicSwapFee(key, swapFee)`

#### 3.1 添加 `beforeSwap` 权限
在 `src/pool-cl/VeCakeDiscountHook.sol` 文件中实现以下内容。Hook 合约扩展自 `CLBaseHook` 和 `ICLDynamicFeeManager`。

```solidity
contract VeCakeDiscountHook is CLBaseHook { // [!code focus]
    function getHooksRegistrationBitmap() external pure override returns (uint16) {
        return _hooksRegistrationBitmapFrom(
            Permissions({
                beforeInitialize: false,
                afterInitialize: false,
                beforeAddLiquidity: false,
                afterAddLiquidity: false,
                beforeRemoveLiquidity: false,
                afterRemoveLiquidity: false,
                beforeSwap: true, // [!code focus]
                afterSwap: false,
                beforeDonate: false,
                afterDonate: false,
                noOp: false
            })
        );
    }
}
```

#### 3.2 在 `beforeSwap()` 中调用 `poolManager.updateDynamicSwapFee(key, swapFee)`
我们在上一步中指定了 `beforeSwap` 权限。因此，CLPoolManager 会在交换之前调用 `hook.beforeSwap` 方法。
现在，我们将实现 `beforeSwap` 方法，调用 `poolManager.updateDynamicSwapFee(key, swapFee)`。

```solidity
function beforeSwap(address, PoolKey calldata key, ICLPoolManager.SwapParams calldata, bytes calldata)
    external
    override
    poolManagerOnly
    returns (bytes4)
{
    uint24 swapFee = getDefaultSwapFee(key);

    // 如果用户是 veCake 持有者，则给予 50% 的折扣。
    if (veCake.balanceOf(tx.origin) >= 1 eth) {
        swapFee = swapFee / 2;
    }

    poolManager.updateDynamicSwapFee(key, swapFee);
    return this.beforeSwap.selector;
}

/// @return 池的默认交换费
function getDefaultSwapFee(PoolKey calldata key) internal view returns (uint24) {
    return key.fee & SwapFeeLibrary.STATIC_FEE_MASK;
}
```
