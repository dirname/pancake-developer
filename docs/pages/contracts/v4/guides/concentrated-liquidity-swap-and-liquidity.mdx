# 浓缩流动性池的流动性和交换

在本指南中，我们将使用 [v4-periphery](https://github.com/pancakeswap/pancake-v4-periphery) 的外围合约

1. 使用 `NonFungiblePositionManager` 进行流动性操作
2. 通过 `CLSwapRouter` 进行交换

<Callout type="info">
  在调用以下任意函数之前，请务必批准 NonFungiblePositionManager 或 SwapRouter。
</Callout>

## 流动性操作

[NonFungiblePositionManager](https://github.com/pancakeswap/pancake-v4-periphery/blob/main/src/pool-cl/NonfungiblePositionManager.sol)

### Mint

使用 NFT 创建一个新的头寸。 如果您之前已经铸造过，可以考虑使用 `increaseLiquidity()` 来向相同的 NFT tokenId 添加流动性。

```solidity
INonfungiblePositionManager.MintParams memory params = INonfungiblePositionManager.MintParams({
  poolKey: key,
  tickLower: tickLower,
  tickUpper: tickUpper,
  amount0Desired: amount0,
  amount1Desired: amount1,
  amount0Min: 0,
  amount1Min: 0,
  recipient: address(this),
  deadline: block.timestamp
});

(uint256 tokenId, uint128 liquidity, uint256 amount0, uint256 amount1)
    = nfp.mint(params);
```

输入：

- `poolKey`：标识池的结构体
- `tickLower | tickUpper`：从 tickLower 到 tickUpper 添加流动性
- `amount0Desired | amount1Desired`：预期的 amount0 和 amount1
- `amount0Min | amount1Min`：最小的 amount0 和 amount1。如果数量低于此值，交易将回滚。在实际应用中，设置一个合理的数值而不是 `0`。
- `recipient`：接收代表该头寸所有权的 NFT 的人
- `deadline`：交易的截止时间。如果 `block.timestamp` 大于 `deadline`，交易将回滚

输出：

- `tokenId`：铸造的 NFT tokenId
- `liquidity`：铸造的总流动性
- `amount0`：用来铸造的 token0 数量
- `amount1`：用来铸造的 token1 数量

### Decrease liquidity
减少头寸中的流动性数量，并将余额记入该头寸

```solidity
INonfungiblePositionManager.DecreaseLiquidityParams memory params = INonfungiblePositionManager
  .DecreaseLiquidityParams({
    tokenId: tokenId,
    liquidity: liquidity,
    amount0Min: 0,
    amount1Min: 0,
    deadline: type(uint256).max
});

(uint256 amount0, uint256 amount1) = nfp.decreaseLiquidity(params);
```

输入：

- `tokenId`：在之前调用 .mint() 时获得的 NFT tokenId
- `liquidity`：要移除的流动性数量
- `amount0Min | amount1Min`：最小的 amount0 和 amount1。如果数量低于此值，交易将回滚
- `deadline`：交易的截止时间。如果 `block.timestamp` 大于 `deadline`，交易将回滚

输出：

- `amount0`：减少流动性后收到的 token0 数量
- `amount1`：减少流动性后收到的 token1 数量

### Increase liquidity
增加头寸中的流动性数量，所需的 token 由 `msg.sender` 支付

```solidity
INonfungiblePositionManager.IncreaseLiquidityParams memory params = INonfungiblePositionManager
  .IncreaseLiquidityParams({
    tokenId: tokenId,
    amount0Desired: amount0,
    amount1Desired: amount1,
    amount0Min: 0,
    amount1Min: 0,
    deadline: block.timestamp
});

(uint128 liquidity, uint256 amount0, uint256 amount1) = nfp.increaseLiquidity(params);
```

输入：

- `tokenId`：在之前调用 .mint() 时获得的 NFT tokenId
- `amount0Desired | amount1Desired`：预期的 amount0 和 amount1
- `amount0Min | amount1Min`：最小的 amount0 和 amount1。如果数量低于此值，交易将回滚
- `deadline`：交易的截止时间。如果 `block.timestamp` 大于 `deadline`，交易将回滚

输出：
- `liquidity`：添加的流动性数量
- `amount0`：为了增加流动性所需的 token0 数量
- `amount1`：为了增加流动性所需的 token1 数量

### Burn

销毁一个 token ID，将其从 NFT 合约中删除。该 token 必须没有流动性并且所有 tokens 已经提取。

```solidity
nfp.burn(uint256 tokenId);
```

## 交换

[CLSwapRouter](https://github.com/pancakeswap/pancake-v4-periphery/blob/main/src/pool-cl/CLSwapRouter.sol)

### Exact Input Single
以 `amountIn` 的一个 token 交换尽可能多的另一个 token

```solidity
ICLSwapRouterBase.V4CLExactInputSingleParams memory params = ICLSwapRouterBase
  .V4CLExactInputSingleParams({
    poolKey: key1,
    zeroForOne: true,
    recipient: address(this),
    amountIn: 1 ether,
    amountOutMinimum: 0,
    sqrtPriceLimitX96: 0,
    hookData: new bytes(0)
});

uint256 amtOut = router.exactInputSingle(params, block.timestamp);
```

输入：

- `poolKey`：标识池的结构体
- `zeroForOne`：如果为 true，交换 token0 为 token1，否则交换 token1 为 token0
- `recipient`：接收输出 token 的地址
- `amountIn`：预期的输入数量
- `amountOutMinimum`：最小输出数量。如果输出数量少于此值，交易回滚
- `sqrtPriceLimitX96`：限制交换时池所达到的价格，帮助减少滑点
- `hookData`：如果设置，将在 beforeSwap 和 afterSwap 中传递
- `deadline`：交易的截止时间。如果 `block.timestamp` 大于 `deadline`，交易将回滚

输出：

- `amountOut`：交换所得 token 的数量

### Exact Input

沿指定路径以 `amountIn` 的一个 token 交换尽可能多的另一个 token

```solidity
bytes32 params;

ISwapRouterBase.PathKey[] memory path = new ISwapRouterBase.PathKey[](2);
path[0] = ISwapRouterBase.PathKey({
  intermediateCurrency: currency1,
  fee: uint24(3000),
  hooks: IHooks(address(0)),
  hookData: new bytes(0),
  poolManager: poolManager,
  parameters: params.setTickSpacing(10)
});

path[1] = ISwapRouterBase.PathKey({
  intermediateCurrency: currency2,
  fee: uint24(3000),
  hooks: IHooks(address(0)),
  hookData: new bytes(0),
  poolManager: poolManager,
  parameters: params.setTickSpacing(10)
});

ICLSwapRouterBase.V4CLExactInputParams memory input = ICLSwapRouterBase
  .V4CLExactInputParams({
    currencyIn: currency0,
    path: path,
    recipient: address(this),
    amountIn: 1 ether,
    amountOutMinimum: 0
});

uint256 amountOut = router.exactInput(input, block.timestamp);
```

输入：

- `currencyIn`：输入 token
- `path`：描述跳过的池的 `PathKey[]` 数组
- `recipient`：接收输出 token 的地址
- `amountIn`：预期的输入数量
- `amountOutMinimum`：最小输出数量。如果输出数量少于此值，交易回滚
- `deadline`：交易的截止时间。如果 `block.timestamp` 大于 `deadline`，交易将回滚

输出：
- `amountOut`：交换所得 token 的数量

### Exact Output Single
以尽可能少的一个 token 交换 `amountOut` 的另一个 token

```solidity
ICLSwapRouterBase.V4CLExactOutputSingleParams memory input = ICLSwapRouterBase
  .V4CLExactOutputSingleParams({
    poolKey: key1,
    zeroForOne: false,
    recipient: address(this),
    amountOut: 1 ether,
    amountInMaximum: 1.1 ether,
    sqrtPriceLimitX96: 0,
    hookData: new bytes(0)
});

uint256 amountIn = router.exactOutputSingle
```

输入：

- `poolKey`：标识池的结构体
- `zeroForOne`：如果为 true，交换 token0 为 token1，否则交换 token1 为 token0
- `recipient`：接收输出 token 的地址
- `amountOut`：预期的输出数量
- `amountInMaximum`：最大输入数量。如果输出数量少于此值，交易回滚
- `sqrtPriceLimitX96`：限制交换时池所达到的价格，帮助减少滑点
- `hookData`：如果设置，将在 beforeSwap 和 afterSwap 中传递
- `deadline`：交易的截止时间。如果 `block.timestamp` 大于 `deadline`，交易将回滚

输出：
- `amountIn`：交换所需的 token 的数量

### Exact Output
沿指定路径（反向）以尽可能少的一个 token 交换 `amountOut` 的另一个 token

```solidity

bytes32 params;

ISwapRouterBase.PathKey[] memory path = new ISwapRouterBase.PathKey[](2);
path[0] = ISwapRouterBase.PathKey({
  intermediateCurrency: currency0,
  fee: uint24(3000),
  hooks: IHooks(address(0)),
  hookData: new bytes(0),
  poolManager: poolManager,
  parameters: params.setTickSpacing(10)
});

path[1] = ISwapRouterBase.PathKey({
  intermediateCurrency: currency1,
  fee: uint24(3000),
  hooks: IHooks(address(0)),
  hookData: new bytes(0),
  poolManager: poolManager,
  parameters: params.setTickSpacing(10)
});

ICLSwapRouterBase.V4CLExactOutputParams memory input = ICLSwapRouterBase
  .V4CLExactOutputParams({
    currencyOut: currency2,
    path: path,
    recipient: address(this),
    amountOut: 1 ether,
    amountInMaximum: 1.1 ether
});

uint256 amountIn = router.exactOutput(input, block.timestamp);
```

输入：

- `currencyOut`：输出 token
- `path`：描述跳过的池的 `PathKey[]` 数组
- `recipient`：接收输出 token 的地址
- `amountOut`：预期的输出数量
- `amountInMaximum`：最大输入数量。如果需要的输入数量更多，交易回滚
- `deadline`：交易的截止时间。如果 `block.timestamp` 大于 `deadline`，交易将回滚

输出：
- `amountIn`：交换所需的 token 的数量
