# 自定义层 | Hook

Hooks 允许开发者通过在生命周期的各个阶段添加回调来定制一个池。在一个 AMM 中，有 5 个关键操作在前 / 后可能出现 10 个回调。这些关键操作包括池初始化、交换、添加流动性、移除流动性和捐赠。

## 使用 hook 初始化一个池

为了使用 hook 初始化一个池，有一些要点需要注意：

1. 部署 hook 合约（不需要地址挖矿），并在 `getHooksRegistrationBitmap` 方法中定义所需的回调。

```solidity
/// @dev 下面指示需要 `beforeSwap` 回调
function getHooksRegistrationBitmap() external pure override returns (uint16) {
  return _hooksRegistrationBitmapFrom(
    Permissions({
      beforeInitialize: false,
      afterInitialize: false,
      beforeAddLiquidity: false,
      afterAddLiquidity: false,
      beforeRemoveLiquidity: false,
      afterRemoveLiquidity: false,
      beforeSwap: true, // 启用 beforeSwap // [!code hl]
      afterSwap: false,
      beforeDonate: false,
      afterDonate: false,
      noOp: false
    })
  );
}
```

2. 初始化池时，在池的关键值中指定 hook 合约，并记得在参数中调用 `hook.getHooksRegistrationBitmap()`，以便在 `PoolKey` 中保存回调。

```solidity 
key = PoolKey({
  currency0: currency0,
  currency1: currency1,
  hooks: hook, // hook 合约地址 
  poolManager: poolManager,
  fee: uint24(3000), // 0.3% 交换费 
  // 参数包含 hook 回调和 tickSpacing: 10
  parameters: bytes32(uint256(hook.getHooksRegistrationBitmap())).setTickSpacing(10)
});

/// 初始化池
poolManager.initialize(key, Constants.SQRT_RATIO_1_1, new bytes(0));
```

## 常见问题

-----

**Q1: Hook 合约可以在任何地址吗？**

Hook 可以部署在任何地址。回调权限要求在 `poolkey.parameters` 中定义且不可变。任何对回调权限的更改将需要一个新的池。

-----

**Q2: 你能详细解释一下 `poolKey.parameters` 吗？**

`poolKey.parameters` 是一个 bytes32 参数，包括所需的 hook 回调，其格式如下：

```solidity 
// 前 16 位用于 hooks 回调 
[0 - 15]: 保留给 hooks 回调

// 其他位可以由每种池类型使用，例如:
[16 - 39] 集中式流动性池的 tick 间隔
[16 - 31] bin 池的 bin 步长
[16 - xx] 其他新的池类型配置
```

例如在 `ICLHooks.sol` 中，如果第 0 位的值为 1，则需要 beforeInitialise 回调。

```solidity 
uint8 constant HOOKS_BEFORE_INITIALIZE_OFFSET = 0;
```

然而，这些都为你更容易实现！例如，要在参数中定义所需的回调，只需覆盖 hooks 并调用 `hooks.getHooksRegistrationBitmap`，如上所述。
