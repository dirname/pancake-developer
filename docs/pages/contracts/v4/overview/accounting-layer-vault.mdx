# Accounting Layer | Vault

在这个架构的核心是
[Vault](https://github.com/pancakeswap/pancake-v4-core/blob/main/src/Vault.sol)，它作为一个不可变的会计层。它维持了一个代币账本，记录了存入或欠下的代币，从而在每次交易结束时，促进了安全高效的结算过程。

## 锁机制

Vault 中的一个关键机制是 **锁机制**。调用者需要先从 Vault 获得一个锁，然后才能与池管理器进行任何操作（交换、流动性或捐献）。

![Vault](/v4/vault.png)

高层次的步骤如下：

1. 通过 `vault.lock()` 从 Vault 获得一个锁，Vault 将回调到调用者 `lockAcquired(data)`

2. 在 `lockAcquired(data)` 中，调用者执行操作，比如交换 `clPoolManager.swap(...)` 或流动性 `clPoolManager.modifyLiquidity(...)`

3. 池管理器通过 `vault.accountPoolBalanceDelta(...)` 通知 Vault 的余额变动

4. 调用者然后需要通过以下四种方法中的一种与 Vault 的余额进行调整：`take(), settle(), mint() 或 burn()`。

```solidity
// 一个四步流程的例子，为了简洁省略了一些代码
// 参考: https://github.com/pancakeswap/pancake-v4-periphery/blob/main/src/pool-bin/BinFungiblePositionManager.sol 
contract BinFungiblePositionManager {
    function addLiquidity(AddLiquidity calldata params) external {
        // 第一步: 获取锁
        vault.lock(..)
    }

    function lockAcquired(bytes calldata rawData) external {
        // 第二步: 与池管理器执行操作
        CallbackData memory data = abi.decode(rawData, (CallbackData));
        (BalanceDelta delta, ) = poolManager.mint(...)

        // 第四步: 调整余额，下面是一个 token0 的例子
        if (delta.amount0() > 0) {
            // 首先将 token0 转移到 Vault 然后调用 vault.settle
            vault.settle(poolKey.currency0, user, uint128(delta.amount0()))
        } else {
            vault.take(poolKey.currency0, user, uint128(-delta.amount0()))
        }
    }
}
```

## 调整余额

每当你执行某些操作时，[poolManager](https://github.com/pancakeswap/pancake-v4-core/blob/main/src/pool-cl/CLPoolManager.sol) 会返回 BalanceDelta，例如 `swap | modifyLiquidity | donate`。它包含 `int128 amount0` 和 `int128 amount1`，这些是需要与你的 Vault 进行 take 或 settle 的值。

### 负余额变动
在这种情况下，Vault 欠调用者代币数量。 

调用者有两种选择：

1. `vault.take(Currency currency, address to, uint256 amount)` - 这将把欠下的代币从 Vault 转移到指定的地址（`address to`）。

2. `vault.mint(Currency currency, address to, uint256 amount)` - 这将把多余的代币存储在 Vault 上，并记入 `address to`。

### 正余额变动
在这种情况下，调用者欠 Vault 代币数量。 

调用者有两种选择：

1. 将代币转移到 Vault 并调用 `vault.settle(Currency token)`

2. 假设调用者之前已经通过 `vault.mint` 将多余的代币存储在 Vault 上，调用者现在可以调用 `vault.burn(Currency currency, uint256 amount)` 来使用那些存储在 Vault 的多余代币。

### 何时使用 `take/mint/settle/burn`？

这取决于你的使用案例，对于大多数使用案例，`take/settle` 就足够了。

不过如果你在开发一个进行大量交易的合约，例如套利机器人，你可以考虑使用 `mint/burn` 来通过减少 ERC-20 代币传输的次数来节省 gas 费用。
