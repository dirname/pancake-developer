# AMM 层 | PoolManager

Pool Managers 是单例合约，封装了不同类型 AMMs 的逻辑。在初始部署中，将有 2 种 AMM 类型，访问它们各自的页面了解更多信息。

1. [CLPoolManager](/contracts/v4/overview/amm-layer/concentrated-liquidity)
2. [BinPoolManager](/contracts/v4/overview/amm-layer/liquidity-book)

### 单例

在 PancakeSwap v3 中，每个池都是一个独立的合约。在 PancakeSwap v4 中，所有池将被封装在一个单一的 PoolManager 合约中。例如，所有集中流动性池在 [CLPoolManager](https://github.com/pancakeswap/pancake-v4-core/blob/main/src/pool-cl/CLPoolManager.sol) 中，以及所有流动性书池在 [BinPoolManager](https://github.com/pancakeswap/pancake-v4-core/blob/main/src/pool-bin/BinPoolManager.sol) 中。

### PoolKey | PoolId

[PoolKey](https://github.com/pancakeswap/pancake-v4-core/blob/main/src/types/PoolKey.sol) 是描述每个池的结构体。

```solidity
/// @notice 返回用于标识池的关键数据
struct PoolKey {
    /// @notice 池中的低阶货币，按数值排序
    Currency currency0;
    /// @notice 池中的高阶货币，按数值排序
    Currency currency1;
    /// @notice 池的钩子，不会有通用接口，因为钩子接口因池类型不同
    IHooks hooks;
    /// @notice 池的池管理器
    IPoolManager poolManager;
    /// @notice 池的交换费，上限为1_000_000。上位 4 位确定钩子是否设置费用。
    uint24 fee;
    /// @notice 钩子的回调和池的特定参数，例如 CL 的 tickSpacing，bin 的 binStep
    bytes32 parameters;
}
```

[PoolId](https://github.com/pancakeswap/pancake-v4-core/blob/main/src/types/PoolId.sol) 是 `keccak256(abi.encode(poolKey))` 的 bytes32 。您会注意到某些来自 PoolManager 的输入需要 PoolId。通过以下方法从 `PoolKey` 派生 `PoolId`：

```solidity
import {PoolIdLibrary} from "@pancakeswap/v4-core/src/types/PoolId.sol";

// 对 PoolKey 类型使用 PoolIdLibrary
using PoolIdLibrary for PoolKey;

PoolKey memory key = PoolKey({
  currency0: currency0,
  currency1: currency1,
  hooks: IHooks(address(0)),
  poolManager: poolManager,
  fee: uint24(3000),
  parameters: bytes32(uint256(counterHook.getHooksRegistrationBitmap())).setTickSpacing(10)
});

// 通过 PoolIdLibrary 从 PoolKey 获取 PoolId
Poolid id = key.toId();
```

### 初始化池

在池上进行交换或流动性操作之前，首先需要初始化池。要初始化，请调用 `CLPoolManager` 或 `BinPoolManager` 的初始化方法。

```solidity
// CLPoolManager
function initialize(PoolKey memory key, uint160 sqrtPriceX96, bytes calldata hookData);

// BinPoolManager
function initialize(PoolKey memory key, uint24 activeId, bytes calldata hookData);
```
