# 集中流动性池

## 概览

集中流动性（来自 UniswapV3）增强了传统的自动做市商（AMM）模型。
它允许流动性提供者（LPs）将资金集中在池内的特定价格区间，而不是像 v2 那样在整个交易范围内均匀分布流动性。

![集中流动性曲线](/v4/amm-layer-poolmanager-cl-1.png)

## 概念

### Tick
Tick 指的是价格区间内的一个特定价格点。池的整个价格范围被划分为离散的区间，每个区间由 `tickLower, tickUpper` 定义。

```solidity 
// 源自 ICLPoolManager.sol
struct ModifyLiquidityParams {
    // 头寸的上下 tick
    int24 tickLower;
    int24 tickUpper;
    // 如何修改流动性
    int256 liquidityDelta;
}

/// @dev 更多信息请见 CLPoolManager
function modifyLiquidity(PoolKey memory key, ICLPoolManager.ModifyLiquidityParams memory params, bytes calldata hookData);
```

以下是根据 tick 计算价格的示例：

```
price = 1.0001 ** tick 
```

#### 示例 1: [CAKE/WETH](https://etherscan.io/address/0x517f451b0a9e1b87dc0ae98a05ee033c3310f046) 池:

```solidity
// token0: CAKE (18 位小数)
// token1: WETH (18 位小数)
// tick: -68554 (截至 2024 年 3 月 12 日)

price: 1.0001 ** -68554 = 0.00105411128
// 这也意味着 1 CAKE = 0.00105411128 WETH 
```

#### 示例 2: [WETH/USDT](https://etherscan.io/address/0x6ca298d2983ab03aa1da7679389d955a4efee15c) 池，带有不同代币小数:

```solidity
// token0: WETH (18 位小数)
// token1: USDT (6 位小数)
// tick: -193256 (截至 2024 年 3 月 12 日)

price: 1.0001 ** -193256 = 4.04965646e-9
// 这也意味着 1 WETH = 4.04965646e-9 USDT 
// 但是 WETH 是 18 位小数，而 USDT 是 6 位小数，我们必须将其乘以 12 位小数
带小数的 WETH/USDT 比率: = 4.04965646e-9 * (10**18/10**6): 1 WETH = 4049.65646 USDT
```

### Tick 间隔

Tick 间隔定义了池内相邻两个 tick 之间的最小价格变动。这是 `PoolKey` 中的一个参数。
请记住，这里存在权衡。较小的 tick 间隔允许更精确的流动性定位，但可能会带来更高的 gas 成本。较大的 tick 间隔在 gas 成本方面可能更便宜，但可能会在交易执行中带来更高的滑点。

| Tick 间隔 | Tick 之间的价格变动 |
| ------ | ------ |
| 1 | 0.01% |
| 10 | 0.1% |
| 50  | 0.5% |
| 100  | 1% |


### sqrtPriceX96

这个 uint160 值表示 token0 和 token1 的平方根价格比率。它是一个二进制定点数 `Q64.96`，即 64 位整数和 96 位小数。

因此
```solidity
sqrtPriceX96 = sqrt(price) * 2**96

// 将价格移到 LHS 结果如下：
price = (sqrtPriceX96 / 2**96) ** 2
```

以下是根据 sqrtPriceX96 计算价格比率的示例：

#### 示例 1: [CAKE/WETH](https://etherscan.io/address/0x517f451b0a9e1b87dc0ae98a05ee033c3310f046) 池:

```solidity 
// token0: CAKE (18 位小数)
// token1: WETH (18 位小数)
// sqrtPriceX96: 2574020484831874748518739167 (截至 2024 年 3 月 12 日)

// token0 (CAKE) 与 token1 (WETH) 的价格比率：
price = (2574020484831874748518739167 / 2**96) **2 = 0.00105551602

CAKE 对 WETH: 1 CAKE = 0.00105551602 WETH 
WETH 对 CAKE: 1 / 0.00105551602: 1 WETH = 947.40 CAKE 

// CAKE 约为 $4.3 而 ETH 为 $4100。所以 947.40 CAKE = 4073 USDC
```

#### 示例 2: [WETH/USDT](https://etherscan.io/address/0x6ca298d2983ab03aa1da7679389d955a4efee15c) 池，带有不同代币小数:

```solidity
// token0: WETH (18 位小数)
// token1: USDT (6 位小数)
// sqrtPriceX96: 5046499860369450237927407 (截至 2024 年 3 月 12 日)

// token0 (WETH) 与 token1 (USDT) 的价格比率: 4.0571e-9 ETH 对 1 USDC
price = (5046499860369450237927407 / 2**96) **2 = 4.0571e-9

WETH 对 USDC: 4.0571e-9 / (10**6/10**18): 1 WETH = 4057 USDT
USDT 对 WETH: 4.0571e-9 / (10**18/10**6): 1 USDT = 4.0571e-21 WETH 
```


### 流动性变化

流动性变化，`int` 值，指的是 `modifyLiquidity` 时的流动性差异。当其为正时，表示用户正在增加流动性，相反，当其为负时，表示用户正在移除流动性。请参考 `LiquidityAmounts.sol` 来计算流动性。
